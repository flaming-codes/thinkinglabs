# Responsive typography and fluid tokens

Typography is centralized in `src/frontend/thinkinglabs-ui/styles.css` under `:root` fluid tokens (`--tl-type-*`).

## Token families

- **Mono/UI**: `--tl-type-mono-2xs`, `--tl-type-mono-xs`, `--tl-type-mono-sm`, `--tl-type-ui-sm`, `--tl-type-ui-md`
- **Body**: `--tl-type-body-sm`, `--tl-type-body-md`, `--tl-type-body-lg`, `--tl-type-body-xl`
- **Display**: `--tl-type-display-xs` through `--tl-type-display-hero-wide`
- **Special**: `--tl-type-row-title`

Each token is fluid (`clamp(...)`) and already tuned for the responsive refactor.

## Usage rules

1. **Always consume tokens via CSS vars**, not hard-coded `px` sizes in components.
2. **Pick by semantic role**, not by visual guess:
   - labels/meta → mono/ui tokens
   - paragraph/prose → body tokens
   - headings/lede/hero → display tokens
3. **Keep type decisions in shared styles** (`styles.css`) so Storybook and frontend stay aligned.
4. **Avoid one-off clamp formulas** in page/component CSS unless adding a reusable token first.

## Current responsive system

- Layout breakpoints are centralized at:
  - `@media (max-width: 1100px)`
  - `@media (max-width: 820px)`
  - `@media (max-width: 620px)`
- Most layout changes at breakpoints adjust grids/gaps/padding, while type keeps scaling through fluid tokens.

## Practical examples

Existing style usage:

```css
.tl-page-head h1 {
  font-size: var(--tl-type-display-hero);
}

.tl-home-thesis-row p {
  font-size: var(--tl-type-body-md);
}

.tl-status-tag {
  font-size: var(--tl-type-mono-2xs);
}
```

When introducing new UI in `src/frontend/thinkinglabs-ui/pages/*` or `components/*`:

- Reuse an existing `--tl-type-*` token first.
- If no token fits, add a new token at the `:root` block in `styles.css`, then reuse it consistently.
- Verify in Storybook across page stories (`Thinkinglabs/Pages/*`) before integrating into `src/pages/**`.

## Storybook ↔ frontend consistency checklist

- `styles.css` is loaded in Storybook via `.storybook/preview.ts`.
- Keep shared UI imports pointed to `src/frontend/thinkinglabs-ui/...`.
- Use the same composition + token system in production routes to avoid drift between Storybook and live pages.
