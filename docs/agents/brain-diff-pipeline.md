# Brain-diff pipeline

The brain-diff feed publishes a stream of meaningful changes across `content/{thoughts,claims,decisions,predictions,projects,posts,observations}`. Two stages: `src/lib/brain-diff.ts` walks `git log --name-status` and `git show`, parses frontmatter once per side, and assigns each file change a deterministic `EntryType` via `classify()` — pattern-match on path prefix and frontmatter delta, no LLM. Specialized brain-diff JSON feeds (`brain-diff-predictions-resolved.json`, `brain-diff-claims-revised.json`, `brain-diff-decisions-reversed.json`) are filtered straight from this stage via `FEED_PREDICATES`. They work with no API key and intentionally do not overwrite the JSON Feed 1.1 files from `pnpm build:feeds`.

`src/lib/brain-diff-score.ts` is the optional second stage. It calls the LLM (OpenAI gpt-4.1-mini, `tier: "fast"` via `src/lib/llm.ts`) once per file change with a forced tool-use response (`extract_substantiveness({ score, summary })`) validated by Zod. The generic feed (`brain-diff.xml`/`brain-diff.json`) drops entries with score < 4; specialized feeds always include their type's matches. Driver: `scripts/brain-diff.ts`. Flags: `--since`, `--out`, `--no-llm`. Use `pnpm brain-diff:offline` or `pnpm artifacts` for offline local generation, and `pnpm brain-diff:scored` or `pnpm artifacts:scored` when the active provider key is set and scored output is required.
